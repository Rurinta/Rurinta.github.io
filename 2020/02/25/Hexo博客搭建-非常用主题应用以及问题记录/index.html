Hexo博客搭建-非常用主题应用以及问题记录
2020-02-25 15:46   label 过程记录, Hexo

19年初打算使用博客来记录日常的学习记录，但最终还是直接使用了CSDN的博客。之后的生活千变万化，再想起来搭建Hexo博客已经是2020年了。疫情影响，在家搞事。

初步搭建hexo博客时使用了推荐比较多的Hexo+Github模式，参考了各种大佬的博文，涉及到的配置已经不太记得了。当时配置结束时候的状态是能够使用Github的地址访问到默认主题的helloword博文。于是这次主要是配置了主题，就可以直接开始写作了。

这篇博文主要是记录一下主题配置的过程，以及非常用（指非NeXT）主题的配置过程中可能会遇到的问题。

主题配置过程

使用的是Makito的Journal.
简洁且优雅完全长在我的审美上。

  • 打开博客文件夹内的themes文件夹

  • 在文件夹内点击“git bash here”（即打开命令行）

  • 找到需要安装主题的Git地址并复制

  • 在命令行窗口内输入“git clone 主题的github地址”，如果是命令行新手，需要知道命令行内不可以使用ctrl+v的快捷键，只能够右键并选择paste

  • 选择txt方式打开博客文件夹内的_config.yml文件，并找到theme:
    字段，将此字段改成主题名称，如果使用Journal.主题，则应该修改为theme: journal

  • 此时若在命令行中分别输入hexo clean+hexo g，则可以在localhost:4000中看见主题应用的效果。

  • 另外，根据主题的不同，在主题文件夹内的_config.yml里可以修改相关的内容。比如博客名称的显示。

做到这一步就已经配置好主题了，但是此时还没有其他的页面，可以使用hexo new page friends来创建友链页面，创建成功后应该就能在网站的侧边栏上看到友链的入口了。

一些问题

搭建hexo博客静态框架教程及操作时踩的坑

这篇博文总结的问题都比较常见，我的文章仅做自己遇到的问题记录，其他的问题可以先参考上面这篇博文。

1 · 字体显示问题

我所使用的主题默认为思源宋体，但在应用的时候时常在网页中看不到思源宋体的显示。

大概琢磨出两个可能的解决方向：

1、js文件中字体文件的显示顺序可能影响

2、更改相关的文字之后需要清除缓存并且重新上传

2 · 运行hexo server报错Cannot read property ‘offset’ of null

这里是自己没注意看error提示了，其实就是timezone设置有问题。

1、检查一下时区设置里的拼写是否正确

2、时区改成Asia/Shanghai是绝对没问题的

3 · WARN babel-eslint@10.0.3 requires a peer of eslint@>= 4.12.1 but none is installed. You must install peer dependencies yourself.

这个报错出在执行npm install hexo-deployer-git --save时。

这一步的目的是下载插件使得后续可以将文件和代码上传托管在github上。报错的意思是前置有组件没有下载，但是在报错的最后会提示：

这个报错其实不是很影响，直接执行后续的hexo命令就好。

但我们也可以按照这个提示输入npm audit fix解。后续可能还会提示要run个啥啥啥去fix，照做即可，如果命令执行的响应时间太长或者执行失败，忽略它就好。

4 · 部署至Github时，原有的README.md文件消失

部署上Github时，仓库里原有的README.md文件会被渲染，按照以下步骤即可使README.md文件绕过渲染：

1、在博客文件夹内的source文件夹内添加一个README.md。

2、修改博客文件夹内的_config.yml。

3、将skip_render参数的值设置为skip_render: README.md
保存并关闭文件。

这样之后再使用hexo d命令，就不会再渲染README.md这个文件了。

到目前为止，在搭建博客时就遇到过这些问题。目前只有写文章记录的需求，之后如果有其他的需求再去慢慢探索。
